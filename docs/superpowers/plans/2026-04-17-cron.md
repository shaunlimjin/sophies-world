# Weekly Cron Job Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Schedule `generate.py && send.py` to run automatically every Saturday at 6am Pacific on the Mac Mini.

**Architecture:** A wrapper shell script (`scripts/run.sh`) sets PATH correctly (so the `claude` CLI is found), runs the two Python scripts in sequence, and appends timestamped output to `logs/run.log`. A single crontab entry calls the script every Saturday at 6am.

**Tech Stack:** bash, macOS cron (`crontab`)

## Task 1: Create `scripts/run.sh` and update `.gitignore`

**Files:**
- Create: `scripts/run.sh`
- Modify: `.gitignore`

- [ ] **Step 1: Create `scripts/run.sh`**

```bash
#!/bin/bash
export PATH="/opt/homebrew/bin:/usr/local/bin:/usr/bin:/bin"
cd /Users/hobbes/dev/sophies-world

mkdir -p logs
echo "--- $(date) ---" >> logs/run.log
/usr/bin/python3 scripts/generate.py >> logs/run.log 2>&1 && \
/usr/bin/python3 scripts/send.py >> logs/run.log 2>&1
echo "Exit: $?" >> logs/run.log
```

- [ ] **Step 2: Make `run.sh` executable**

```bash
chmod +x /Users/hobbes/dev/sophies-world/scripts/run.sh
```

- [ ] **Step 3: Add `logs/` to `.gitignore`**

Open `/Users/hobbes/dev/sophies-world/.gitignore` and add `logs/` so it reads:

```
.env
__pycache__/
*.pyc
.pytest_cache/
logs/
```

- [ ] **Step 4: Commit**

```bash
cd /Users/hobbes/dev/sophies-world
git add scripts/run.sh .gitignore
git commit -m "feat: add run.sh wrapper script and gitignore logs/"
```

## Task 2: Set up the crontab entry

**Files:**
- System crontab (edited via `crontab -e`)

- [ ] **Step 1: Verify the Mac Mini's timezone is Pacific**

```bash
sudo systemsetup -gettimezone
```

Expected output: `Time Zone: America/Los_Angeles`

If it shows something else, set it:
```bash
sudo systemsetup -settimezone America/Los_Angeles
```

- [ ] **Step 2: Open crontab for editing**

```bash
crontab -e
```

This opens your user crontab in `vi`. Press `i` to enter insert mode.

- [ ] **Step 3: Add the cron entry**

Add this line:

```
0 6 * * 6 /Users/hobbes/dev/sophies-world/scripts/run.sh
```

`0 6 * * 6` means: minute 0, hour 6, any day of month, any month, Saturday (6). Press `Esc`, then type `:wq` and press Enter to save.

- [ ] **Step 4: Verify the entry was saved**

```bash
crontab -l
```

Expected output includes:
```
0 6 * * 6 /Users/hobbes/dev/sophies-world/scripts/run.sh
```

## Task 3: Smoke test

- [ ] **Step 1: Run `run.sh` manually**

```bash
/Users/hobbes/dev/sophies-world/scripts/run.sh
```

Expected: script runs, appends to `logs/run.log`. Since today's newsletter already exists, `generate.py` will print "Output already exists, skipping" and `send.py` will either send or (if it was already sent today) send again — that's fine for a test.

- [ ] **Step 2: Check the log**

```bash
cat /Users/hobbes/dev/sophies-world/logs/run.log
```

Expected output like:
```
--- Thu Apr 17 06:00:00 PDT 2026 ---
Output already exists, skipping: .../newsletters/sophies-world-2026-04-17.html
Sending Issue #2 to sophie.lim.xin.en@gmail.com...
Sent!
Exit: 0
```

- [ ] **Step 3: Grant Full Disk Access to cron if needed**

If the manual run worked but the cron job silently fails, macOS may be blocking it:

1. Open **System Settings → Privacy & Security → Full Disk Access**
2. Click `+` and add `/usr/sbin/cron`
3. Restart the cron daemon: `sudo launchctl stop com.vix.cron && sudo launchctl start com.vix.cron`

This only needs to be done once.
